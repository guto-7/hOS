### Formatting Raw Data

The final step of Importing. Once a node has defined what data it accepts and what it rejects, it must determine how that raw data is extracted and preserved before handoff to the Unifying layer. This is not transformation or interpretation — it is getting raw input into a state the pipeline can work with.

---

### Why Formatting Decisions Matter Early

How data is formatted at this stage directly constrains what is possible downstream. An image stored at the wrong resolution or colour space limits what machine learning models can extract from it. A PDF extracted as flat text with no structural metadata makes pattern matching in the parser unreliable. Formatting decisions cannot be made in isolation — they require a general understanding of what the Evaluation layer (Layer 3) will need.

This does not mean Importing needs to know the full evaluation strategy. It means the person defining this layer must ask: *what does the data need to look like for deterministic analysis to be possible?*

---

### What Formatting Covers

##### Each node must define:

**Extraction Method**
- How raw data is pulled from the source format. PDF text extraction, DICOM pixel data access, manual entry field capture, etc.
- What tooling or approach is used and why.

**Preservation Format**
- What intermediate format the extracted data is stored in before Unifying.
- This should retain as much of the original data as possible. Lossy extraction at this stage is permanent — anything discarded here cannot be recovered later.

**Structural Metadata**
- What contextual information is preserved alongside the raw data. Page structure, field labels, image dimensions, acquisition parameters, timestamps, source identifiers.
- Metadata that seems irrelevant at Importing may be critical at Evaluation. Default to preserving it.

**Downstream Alignment**
- Document how formatting choices account for the needs of later layers — specifically Unifying (Layer 2) and Evaluation (Layer 3).
- If the Evaluation layer requires a specific data structure or resolution, formatting must deliver it or explicitly document why it cannot.

---

## Node Implementation

When applying this framework to a specific node, document the following in that node's Importing section:

###### Extraction Method
- Define how raw data is extracted from each accepted file format, the tooling used, and any known limitations of that extraction approach.

###### Preservation Format
- Define the intermediate format and what data is retained. Document any unavoidable data loss and its impact on downstream layers.

###### Structural Metadata
- List the metadata fields preserved alongside extracted data and why each is included.

###### Downstream Dependencies
- Document what the Unifying and Evaluation layers require from this stage and how those requirements are met.
