### Domain Analysis

Holistic assessment of the diagnostic medium as a whole. Where General Analysis organises data into groups and surfaces critical flags, Domain Analysis produces composite metrics that give a high-level read on overall status. These are not individual flags or condition diagnoses — they are aggregate scores that tell the user how a broad area of their health is performing.

---

### What Domain Analysis is

Composite metrics derived from multiple data points within a grouping or across groupings. For blood work, this might be a metabolic health index derived from glucose, insulin, HbA1c, and triglycerides. For body composition, this might be a cardiovascular risk score derived from waist-to-hip ratio, body fat percentage, and visceral fat estimates.

These metrics compress complexity into actionable signals. A user may not know what their individual lipid markers mean in combination — but a validated cardiovascular risk score communicates status immediately.

---

### What Domain Analysis Is Not

- **Not invented metrics.** Every composite score must be derived from a validated, published scoring system or clinical index. If no validated system exists for a particular evaluation, it does not belong in this layer.
- **Not probabilistic.** Scores are computed deterministically from the data points and the defined formula. Same inputs, same score, every time.
- **Not diagnostic.** A poor metabolic health score is not a diagnosis of metabolic syndrome — that is Condition Mapping's role. Domain Analysis indicates status; Condition Mapping identifies conditions.

---

### What Each Node Must Define

##### Evaluation Selection

- What composite metrics are relevant to this node's diagnostic medium? This requires research into established clinical scoring systems, health indices, and validated assessment tools used in the domain.
- Each selected evaluation must be justified — why is this metric clinically meaningful? What does it tell the user that individual flags or groupings do not?
- Document competing or alternative scoring systems where they exist, and the rationale for choosing one over another.

##### Composition

- What data points feed into each evaluation? The exact formula, algorithm, or scoring rubric must be documented.
- How are missing data points handled? If a score requires five inputs and only four are present, is the score computed with a caveat, or withheld entirely? This intersects with [Diagnostic Certainty](4.%20Diagnostic%20Certainty.md).

##### Diagnostic Method

- What computational approach produces each score? Weighted formulas, lookup tables, regression models, established clinical calculators — the method must be documented alongside the clinical research that validates it.
- If a validated scoring system defines its own calculation method, that method is used as published. If adaptation is required (e.g., different available inputs), the adaptation and its impact on validity must be documented.

##### Storage

- All computed scores and their constituent inputs must be stored in a structured, scalable format. The raw score, the inputs used, and the scoring system version should all be persisted — allowing scores to be recomputed if the methodology is updated and easily formatted for any downstream consumer.

---

### Node Implementation

When applying this framework to a specific node, document the following in that node's Evaluation section:

###### Selected Evaluations
- List each composite metric the node produces. For each: what it measures, the validated scoring system it is based on, and why it was selected.

###### Scoring Definitions
- For each evaluation, document the exact inputs, formula or algorithm, and how missing inputs are handled.

###### Diagnostic Method
- Document the computational approach for each score and the clinical research that validates it.

###### Storage Format
- Define how scores and their constituent data are stored — structure, fields, and versioning approach.

###### Sources
- Reference the clinical literature, published scoring systems, and validation studies for each evaluation.
