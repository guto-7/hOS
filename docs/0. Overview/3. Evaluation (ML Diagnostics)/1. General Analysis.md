### General Analysis

The first pass over evaluated data. Before condition mapping or domain-level scoring, the node must surface what is immediately important and organise the data into meaningful groups. This gives the user a structured overview of their results and provides the foundation that Domain Evaluations and Condition Mapping build on.

---

### Critical Flags

The most urgent findings come first. Any data point flagged at a critical severity level during Initial Flagging (Layer 2) must be surfaced immediately and prominently. These are values that may require urgent attention regardless of what broader patterns they belong to.

General Analysis does not interpret why a value is critical — that is Condition Mapping's role. It ensures nothing critical is buried beneath grouped views or composite scores.

---

### Domain Groupings

All data points must be organised into domain-specific categories that reflect how the diagnostic medium is clinically structured. For blood work, this means panels — hormonal, metabolic, lipid, hepatic, haematological, etc. For imaging, this means anatomical regions or system groupings. For body composition, this means measurement categories.

##### Each node must define:

**What the groupings are**
- The categories that data points are organised into. These are not arbitrary — they reflect established clinical practice for how this medium's data is reviewed and interpreted.

**Why these groupings**
- Research into how the diagnostic medium is structured in clinical literature. Which groupings are standard? Are there competing classification systems? Which is most appropriate for this node's scope?

**What each grouping contains**
- A defined list of which canonical data points belong to each group. Every data point must belong to at least one group. Some may belong to multiple groups where clinically appropriate.

**How groupings are presented**
- Each group should provide a summary view — how many data points are flagged, the overall status of that group, and what stands out. This feeds directly into the node's dashboard UI.

---

### Diagnostic Method

The computational approach for General Analysis is primarily organisational — sorting, prioritising, summarising. However, the logic for determining critical flag priority and group-level summaries must still be defined and documented. If a group summary uses any aggregation logic beyond simple flag counting (e.g., weighting certain data points higher within a group), the method and rationale must be explicit.

---

### Node Implementation

When applying this framework to a specific node, document the following in that node's Evaluation section:

###### Critical Flag Criteria
- Define which severity levels from Initial Flagging trigger critical surfacing in General Analysis. Document any additional criteria beyond severity level.

###### Domain Groupings
- Define the complete set of groupings for this node's medium. For each group: what it contains, the clinical basis for the grouping, and which data points it includes.

###### Group Summaries
- Define how each group's overall status is calculated — flag counts, weighted summaries, or other aggregation methods. Document the logic and rationale.

###### Sources
- Reference the clinical literature, classification systems, or established standards that informed the grouping structure.
