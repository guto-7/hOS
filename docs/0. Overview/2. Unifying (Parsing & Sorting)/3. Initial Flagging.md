### Initial Flagging

The final step of Unifying. Once data is standardised and validated, each individual data point is annotated against its known reference. This is not diagnostic analysis — it is per-value annotation that marks where a data point sits relative to established norms.

---

### What Initial Flagging Is

A single data point compared to a single reference. For blood work: a marker value checked against its reference range. For imaging: a measurement checked against established anatomical norms. For body composition: a metric checked against population-based standards.

The output is a flag attached to the data point — normal, high, low, critical, optimal, or whatever classification system the node defines. Nothing more. No pattern recognition, no cross-referencing, no condition matching. Those are Evaluation's responsibility.

---

### What Initial Flagging Is Not

- **Not diagnostic.** A high LDL flag does not mean cardiovascular risk. A below-range organ measurement does not mean pathology. Flags describe where a value sits — Evaluation determines what it means.
- **Not multi-variable.** Each data point is flagged independently. Relationships between data points are not considered here.
- **Not probabilistic.** Flagging is deterministic. The same value against the same reference produces the same flag every time.

---

### What Each Node Must Define

##### Reference Standards

- What reference data is each data point compared against? Clinical reference ranges, anatomical norms, population standards — the source must be documented and evidence-based.
- References may vary by demographic factors (age, sex, ethnicity, pregnancy status). The node must define which factors adjust the reference and how.

##### Flag Classifications

- What categories does this node use? A simple HIGH/NORMAL/LOW scheme may be sufficient for some domains. Others may need finer granularity — CRITICAL_HIGH, ABOVE_OPTIMAL, BORDERLINE, etc.
- The classification system must be defined once for the node and applied consistently. Every data point uses the same set of flag categories.

##### Quantified Severity

- Flags must convey magnitude, not just direction. A value 5% above range and a value 300% above range are fundamentally different clinical signals.
- **Deviation metric** — each node must define how distance from the reference boundary is quantified. The appropriate metric depends on the domain and data type:
  - Percentage deviation from the upper or lower bound.
  - Standard deviations from the mean for normally distributed values.
  - Multiples of the reference limit (e.g., xULN for hepatic markers).
  - Graded scoring systems for non-numeric data (e.g., imaging classification scales like BI-RADS for radiology, morphological grading systems).
  - Composite metrics where a single data point has multiple measurable dimensions (e.g., a structural measurement that deviates in both size and density).
- Not all data is a single number on a scale. Some domains produce categorical, spatial, or multi-dimensional data — the severity metric must fit the data, not the other way around.
- **Severity thresholds** — defined per data point, not globally. What constitutes a critical deviation for one data point may be clinically insignificant for another.
- **Threshold justification** — severity boundaries must be derived from clinical literature or established classification systems, not arbitrary cutoffs. Document the source and rationale for each threshold.

---

## Node Implementation

When applying this framework to a specific node, document the following in that node's Unifying section:

###### Reference Sources
- For each canonical data point, define the reference standard used, its source, and any demographic adjustments.

###### Flag Categories
- Define the complete set of flag classifications for this node. Document what each category means and where the boundaries sit.

###### Severity Quantification
- Define the deviation metric(s) used for this node's data points — percentage deviation, standard deviations, multiples of reference limits, classification scales, or domain-specific measures. Document why the chosen metric is appropriate for the data type.
- For each canonical data point, define the severity thresholds and the clinical literature or classification systems that support them.

###### Edge Cases
- Document how borderline values are handled — values that sit exactly on a reference boundary or fall between tiers.
